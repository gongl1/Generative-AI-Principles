6.1 Different categories of finetuning¶

  • No code in this section
  • The most common ways to finetune language models are instruction-finetuning and classification finetuning
  • Instruction-finetuning, depicted below, is the topic of the next chapter
  • Classification finetuning, the topic of this chapter, is a procedure you may already be familiar with if you have a background in machine learning -- it's similar to training a convolutional network to classify handwritten digits, for example
  • In classification finetuning, we have a specific number of class labels (for example, "spam" and "not spam") that the model can output
  • A classification finetuned model can only predict classes it has seen during training (for example, "spam" or "not spam"), whereas an instruction-finetuned model can usually perform many tasks
  • We can think of a classification-finetuned model as a very specialized model; in practice, it is much easier to create a specialized model than a generalist model that performs well on many different tasks

6.3 Creating data loaders¶

  • Note that the text messages have different lengths; if we want to combine multiple training examples in a batch, we have to either

    1. truncate all messages to the length of the shortest message in the dataset or batch
    2. pad all messages to the length of the longest message in the dataset or batch

  • We choose option 2 and pad all messages to the longest message in the dataset

  • For that, we use <|endoftext|> as a padding token

import torch
from torch.utils.data import Dataset


class IMDBDataset(Dataset):
    # max_length: Optional, specifies the maximum sequence length for all texts. If not provided, the class calculates the longest sequence and uses it as the max_length.
    def __init__(self, csv_file, tokenizer, max_length=None, pad_token_id=50256):
        self.data = pd.read_csv(csv_file)

        # Pre-tokenize texts
        # Encodes each text string using the provided tokenizer, converting each text into a list of token IDs.
        # The resulting self.encoded_texts is a list where each item is a list of integers.
        self.encoded_texts = [
            tokenizer.encode(text) for text in self.data["text"]
        ]

        if max_length is None:
            self.max_length = self._longest_encoded_length()
        else:
            self.max_length = max_length
            # Truncate sequences if they are longer than max_length
            self.encoded_texts = [
                encoded_text[:self.max_length]
                for encoded_text in self.encoded_texts
            ]


        # Pad sequences to the longest sequence
        # Pads each encoded sequence to self.max_length by appending pad_token_id as many times as needed.
        self.encoded_texts = [
            encoded_text + [pad_token_id] * (self.max_length - len(encoded_text))
            for encoded_text in self.encoded_texts
        ]

    # This method allows access to individual data points by their index.
    # encoded: Retrieves the encoded (and padded/truncated) token list for the text at the specified index.
    # label: Retrieves the corresponding label from self.data.
    # Converts both the encoded sequence and label into PyTorch tensors for compatibility with PyTorch models.
    def __getitem__(self, index):
        encoded = self.encoded_texts[index]
        label = self.data.iloc[index]["label"]
        return (
            torch.tensor(encoded, dtype=torch.long),
            torch.tensor(label, dtype=torch.long)
        )


    # Returns the total number of samples in the dataset, equivalent to the number of rows in the loaded CSV.
    def __len__(self):
        return len(self.data)

    # Loops through each encoded text sequence to find the longest sequence length.
    # Updates max_length to the longest observed sequence length,
    # which is then used for padding all sequences to a consistent length if max_length was not provided during initialization.
    def _longest_encoded_length(self):
        max_length = 0
        for encoded_text in self.encoded_texts:
            encoded_length = len(encoded_text)
            if encoded_length > max_length:
                max_length = encoded_length
        return max_length

The IMDBDataset class loads data from the CSV files, tokenizes the text using the GPT-2 tokenizer from tiktoken, and allows us to pad or truncate the sequences to a uniform length determined by either the longest sequence or a predefined maximum length. This ensures each input tensor is of the same size, which is necessary to create the batches in the training data loader we implement

By increasing the batch_size in your DataLoader, you reduce the total number of batches (i.e., len(train_loader)) per epoch. Here’s a quick breakdown:

  • Increasing the batch_size reduces the number of iterations needed to complete an epoch, as more data samples are processed in each batch.

  • For example, if you have 10,000 samples in your dataset, a batch_size of 8 results in 1,250 steps per epoch, whereas a batch_size of 128 would reduce it to about 78 steps per epoch.

You can adjust batch_size to a larger value like 128 (or whatever your GPU memory allows) to reduce the training time by lowering the number of steps per epoch. This will affect the frequency of your print statements if you continue to use eval_freq.

The calc_accuracy_loader function calculates the accuracy of a model over a specified number of batches from a data loader. It evaluates the model's performance without updating the model's parameters, which is standard practice when measuring accuracy on validation or test datasets.

The function iterates through a specified number of batches, collects the model’s predictions, and calculates the accuracy by comparing predictions with true labels. The function uses eval() mode and torch.no_grad() to ensure efficient and accurate evaluation without modifying the model’s parameters.

num_batches: Optional, specifies the maximum number of batches to use for the accuracy calculation. If None, it uses all batches in data_loader.

def calc_accuracy_loader(data_loader, model, device, num_batches=None):

    # model.eval(): Sets the model to evaluation mode, which disables dropout and other training-specific behaviors.
    # correct_predictions and num_examples are counters used to track the number of correct predictions and the total number of examples processed, respectively.
    model.eval()
    correct_predictions, num_examples = 0, 0

    if num_batches is None:
        num_batches = len(data_loader)
    else:
        num_batches = min(num_batches, len(data_loader))

    # Loops through each batch, up to num_batches.
    # input_batch and target_batch are the input data and corresponding labels for the current batch, and they’re transferred to the specified device.
    for i, (input_batch, target_batch) in enumerate(data_loader):
        if i < num_batches:
            input_batch, target_batch = input_batch.to(device), target_batch.to(device)

            # torch.no_grad(): Disables gradient calculations for efficiency since gradients aren’t needed for evaluation.
            # logits = model(input_batch)[:, -1, :]: Passes input_batch through the model and takes only the logits (predicted scores) for the last token in each sequence (if this is a sequence classification task).
            # predicted_labels = torch.argmax(logits, dim=-1): For each output in the batch, selects the class with the highest score as the predicted label.
            with torch.no_grad():
                logits = model(input_batch)[:, -1, :]  # Logits of last output token
            predicted_labels = torch.argmax(logits, dim=-1)

            # num_examples is updated to reflect the total number of predictions made so far.
            # correct_predictions increments by the count of correct predictions in the current batch.
            # (predicted_labels == target_batch) creates a boolean tensor where True indicates a correct prediction. .sum().item() converts this to a count of correct predictions.
            num_examples += predicted_labels.shape[0]
            correct_predictions += (predicted_labels == target_batch).sum().item()
        else:
            break
    return correct_predictions / num_examples

The calc_loss_loader function calculates the average loss over a specified number of batches from a data loader. It’s designed to evaluate the performance of a model by computing the cross-entropy loss over the input data, typically used in validation or testing scenarios.

  • data_loader: A PyTorch DataLoader that provides batches of input and target data.
  • model: The model to evaluate.
  • device: The device (CPU or GPU) to transfer data and the model.
  • num_batches: Optional, specifies the maximum number of batches to use for the loss calculation. If None, it uses all batches in data_loader.

def calc_loss_loader(data_loader, model, device, num_batches=None):

    # total_loss is a counter to accumulate the total loss across batches.
    total_loss = 0.
    if len(data_loader) == 0:
        return float("nan")

    elif num_batches is None:
        num_batches = len(data_loader)
    else:
        # Reduce the number of batches to match the total number of batches in the data loader
        # if num_batches exceeds the number of batches in the data loader
        # If num_batches is provided, it’s set to the minimum of the specified num_batches and the total number of batches in data_loader.
        # This ensures that num_batches does not exceed the available data.
        num_batches = min(num_batches, len(data_loader))

    # Loops through each batch, up to num_batches.
    # For each batch:
    # Calls calc_loss_batch(input_batch, target_batch, model, device) to calculate the loss for that batch.
    # Adds the scalar value of the batch loss to total_loss using loss.item().
    # calc_loss_batch is a helper function that calculates the cross-entropy loss for a single batch:
    for i, (input_batch, target_batch) in enumerate(data_loader):
        if i < num_batches:
            loss = calc_loss_batch(input_batch, target_batch, model, device)
            total_loss += loss.item()
        else:
            break

    # Divides total_loss by num_batches to get the average loss across the specified batches.
    return total_loss / num_batches

The train_classifier_simple function trains a classifier model on a dataset using a specified optimizer, DataLoaders, and a set number of epochs. It tracks and evaluates the training and validation losses and accuracies at regular intervals.

  • eval_freq: Frequency (in steps) at which to evaluate the model on train and validation data.
  • eval_iter: Number of iterations (batches) to use for each evaluation step.

def train_classifier_simple(model, train_loader, val_loader, optimizer, device, num_epochs,
                            eval_freq, eval_iter):
    # Initialize lists to track losses and examples seen
    train_losses, val_losses, train_accs, val_accs = [], [], [], []

    # examples_seen: Tracks the total number of examples processed, useful for tracking progress.
    # global_step: Counter that tracks the current training step globally across all epochs.
    examples_seen, global_step = 0, -1

    # Main training loop
    for epoch in range(num_epochs):
        # The main loop iterates over each epoch. Setting model.train() enables training-specific operations like dropout and batch normalization.
        model.train()  # Set model to training mode

        for input_batch, target_batch in train_loader:
            # Clears the gradients to prevent accumulation from previous iterations.
            optimizer.zero_grad() # Reset loss gradients from previous batch iteration
            # Calculates the loss for the current batch by comparing predictions to target_batch.
            loss = calc_loss_batch(input_batch, target_batch, model, device)
            # Computes gradients for all trainable parameters based on the loss.
            loss.backward() # Calculate loss gradients
            # Updates the model’s weights using the calculated gradients.
            optimizer.step() # Update model weights using loss gradients
            # examples_seen is incremented by the batch size (number of examples in input_batch).
            examples_seen += input_batch.shape[0] # New: track examples instead of tokens
            # global_step increments by 1 for each batch processed, providing a cumulative count across epochs.
            global_step += 1


            # eval_freq
            # Optional evaluation step
            # This evaluation step runs every eval_freq steps, allowing periodic monitoring of the model’s progress.
            if global_step % eval_freq == 0:
                # evaluate_model(...): Calculates the loss for the training and validation sets using eval_iter batches.
                # train_loss and val_loss represent the average losses for the training and validation sets over eval_iter batches.
                train_loss, val_loss = evaluate_model(
                    model, train_loader, val_loader, device, eval_iter)

                # Appends train_loss and val_loss to their respective lists and prints out the results, providing a checkpoint of the model’s performance.
                train_losses.append(train_loss)
                val_losses.append(val_loss)
                print(f"Ep {epoch+1} (Step {global_step:06d}): "
                      f"Train loss {train_loss:.3f}, Val loss {val_loss:.3f}")



        # Calculate accuracy after each epoch
        # After completing each epoch, the function calculates accuracy on the entire training and validation sets (or up to eval_iter batches if specified).
        # calc_accuracy_loader(...): Computes accuracy by comparing model predictions to actual labels.
        train_accuracy = calc_accuracy_loader(train_loader, model, device, num_batches=eval_iter)
        val_accuracy = calc_accuracy_loader(val_loader, model, device, num_batches=eval_iter)
        print(f"Training accuracy: {train_accuracy*100:.2f}% | ", end="")
        print(f"Validation accuracy: {val_accuracy*100:.2f}%")
        train_accs.append(train_accuracy)
        val_accs.append(val_accuracy)

    return train_losses, val_losses, train_accs, val_accs, examples_seen

def classify_review(text, model, tokenizer, device, max_length=None, pad_token_id=50256):
    model.eval()

    # Prepare inputs to the model
    input_ids = tokenizer.encode(text)
    supported_context_length = model.pos_emb.weight.shape[0]
    # Note: In the book, this was originally written as pos_emb.weight.shape[1] by mistake
    # It didn't break the code but would have caused unnecessary truncation (to 768 instead of 1024)

    # Truncate sequences if they too long
    input_ids = input_ids[:min(max_length, supported_context_length)]

    # Pad sequences to the longest sequence
    input_ids += [pad_token_id] * (max_length - len(input_ids))
    input_tensor = torch.tensor(input_ids, device=device).unsqueeze(0) # add batch dimension

    # Model inference
    with torch.no_grad():
        logits = model(input_tensor)[:, -1, :]  # Logits of the last output token
    predicted_label = torch.argmax(logits, dim=-1).item()

    # Return the classified result
    return "positive" if predicted_label == 1 else "negative"
